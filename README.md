org-pomodoro中だと動画を取るレコーダーと、ビューワー。

## GOAL

リポジトリを始めるときにいろんな既存リポジトリからコピペしていて不便である。シンプルなアプリケーションを題材に、ほかのプロジェクトで流用しやすくする。

望ましいと考える状態と達成状況を書く。

- バックエンド(アプリケーション)
  - [x] 先にAPIスキーマを変更しないと実装を変更できない
  - [ ] 構造化ログ、ログレベルが設定されている
  - [x] OpenAPI仕様に一致してバリデーションできる(ミドルウェアが設定されている)
  - [x] エラー内容とレスポンスHTTPコードの関連付けが設定されており、500エラーは本当に必要なときしか出ない
  - [x] アプリケーション設定を環境変数で行える。コード内で設定を変数として扱える
  - [ ] seedが保守しやすい、漏れに気付ける。設定が柔軟でさまざまな環境をコードで再現しやすい
  - [ ] マイグレーションが実行できる、マイグレーションを自動テストできる
  - [x] システムでのタイムゾーンはすべてUTCを使用する(データベース/API仕様/ログ...)
- バックエンド(テスト)
  - [ ] 時間を注入できる
  - [x] テストが並列実行できる
  - [ ] 分割して複数マシンで実行できる
  - [x] DBレコードのファクトリ関数が整備されていて作成しやすい
- フロントエンド
  - [x] コンポーネントが適度に分割されている
  - [ ] ユニットテストと、E2Eテストが整備されている
  - [x] UIライブラリを使って、なるべくスタンダードになっているUIに近づける
  - [x] クライアントのロケールを尊重してタイムゾーン/時刻フォーマットを表示する
  - [ ] テストが並列実行できる
- ツール類
  - [x] tblsが整備されていて、DBスキーマの変更に気づける。ドキュメンテーションの漏れに気づける
  - [x] OpenAPI / typespecが整備されていて、APIスキーマを共有しやすい
  - [x] OpenAPIコード生成が整備されていて、APIスキーマと実装との差分が出ない
  - [x] 費用対効果の高いLinterが設定されている
  - [x] フォーマッターが設定されている
- 開発環境
  - [x] Dockerコンテナで開発環境を立ち上げられる
  - [ ] 各アプリケーションの本番向けイメージをpushできる
- CI
  - [x] Github Actionsを使ってコード生成をチェックし、漏れを検知できる
  - [x] Github Actionsを使ってコードをチェックし、品質を維持できる
